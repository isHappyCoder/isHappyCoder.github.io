集合体系

List

List的集合的特点：List集是有序(存入和取出的顺序一致)的，可以重复的。

List中的比较方法都是依赖于被比较对象的的equals()方法。但是自定义的类的equals(),方法的比较都是对象创建的地址，因此如果想要按照自己的逻辑来比较自定义的对象，就需要按照自己的比较逻辑重写equals();

​ List和Vector的区别：

​ List是异步的线程不安全，但是速度快

​ Vector是同步的线程安全，速度慢。

ArrayList

​ ArrayList 的底层实现原理是可变长度的数组，数组数据结构的特点是访问元素快，插入元素，删除元素慢，因为数组具有索引，每次增加或删除元素都要将被删除元素的后面的所有元素向前或向后移动，所以数组的删除和增加会慢。访问时可以直接根据索引访问，访问的速度会快写

ArrayList的创建规率

​ ArrayList 的底层是数组，初次创建时容量为10的空列表存储数据，当这10个空间满了之后，会再创建一个列表，新创建的列表的长度会比前一个列表长50%，如果第一个列表的长度是10，第二个列表的长度就是15.当前一个列表满之后就会创建第二个，并且会将前一个列表的数据复制到新创建的列表中，并清空前一个列表。

LinkedList

LinkedList的底层是链表结构。链表结构的特点就是，访问元素慢，添加和删除快。

Set集合

HashSet

​ HashSet 的底层是哈希表的结构，List集合存储的是对象的引用地址，而HashSet与List不同，存储的是对象的哈希值，如果存储的两个对象的哈希值相同，则继续使用对象的equals(),方法比较两个对象的地址是否相同(即对比两个对象是否相同)，如果相同则不加入集合，如果哈希值相同，而对象不是同一个对象，则在哈希表的同一个位置顺延存储两个不同的哈希值

HashSet判断存入的元素是否相同的依据是：对象的 哈希值相同，且对象的地址也得相同(equals对比)，两者都满足，HashSet认定两个元素相同。如果哈希值不同则认为两个元素不相同，不继续比较地址

判断集合中是否包含某个元素的原理，与判断存入的元素是否相同的依据是一样的

TreeSet

​ TreeSet的使用特点：TreeSet 是set体系中可以实现排序的集合，向TreeSet中存入的数据必须提供排序依据（只是存入一个元素的情况除外，只有一个元素不需要比较排序），实现排序依据的方法是实现Comparable接口，复写int compareTo(T o)方法，在compareTo(T o)中定义自己需要的排序规则。加入到TreeSet中的数据就会按照定义排序规则进行排序。compareTo(T o)的返回值代表：

负整数 当前对象小于指定对象

零 当前对象等于指定对象

正整数 当前对象大于指定对象。

同理在TreeSet集合中判断集合是否包含某个元素，也是依赖的compare(T o),

TreeSet的排序会根据int compare(Object o1, Object o2)或 int compareTo(Object obj)；的返回值来判断原不俗的大小。

也就是说想要把某个对象存入TreeSet，这个对象要么实现Comparable接口重写int compareTo(T o),要么TreeSet在初始化时自己定义比较器，这两个条件必须满足其一,否则在向TreeSet中存入个数超过两个对象是就会报错。

比较器

​ 使用比较器的场景，当自定义的对象中的compareTo(T o )中排序逻辑不满足需求的时候，或者对象中没有定义排序规则，那么就需要我们自己定义比较器来完成比较，比较器是定义在TreeSet 集合中的，当比较器和对象中的compareTo(T o ),同时存在是，对比时会优先使用比较器。比较器的使用方法。
